## Overview

In previous lessons, you completed market research and user research. Now you'll write **dual PRDs** - one for humans (stakeholders) and one for agents (automation systems).

- **Human PRD (Markdown):** For PMs, engineers, designers, executives - readable documentation
- **Agent PRD (JSON):** For automation - ticket writers, test generators, deployment agents - machine-readable specifications

## PART A: Generate Human-Readable PRD

### Step 3: Generate Comprehensive Human PRD

Now let's write a comprehensive human PRD using context chaining and specific prompting:

**Prompt:**
```
Write comprehensive PRD for Lease Compliance Reporting feature.

Context sources to use:
- @outputs/market-research.md for market opportunity, trends, competitive analysis
- @outputs/user-research.md for user personas, pain points, interview insights
- @company-context/ for company strategy, positioning, OKRs
- @templates/prd-template-redesigned.md for structure

Follow the template structure BUT customize with:
- Custom section: "Regulatory Compliance Metrics" (audit accuracy rate, compliance reporting time reduction, supported accounting standards like IFRS and GAAP)

Required sections:
1. Problem statement with market sizing
2. User personas (Legal Team Lead, Compliance Officer, CFO)
3. User stories with acceptance criteria and priority (P0/P1/P2)
4. Feature requirements (functional + non-functional)
5. End-to-end user journey: Jennifer (Legal Team Lead, 150 leases, audit in 10 days) - step-by-step from upload to auditor delivery
6. Non-functional requirements table: performance, security, data retention, availability
7. Technical specifications: data models (Lease, ComplianceReport, AuditTrail), API specs, integration points (QuickBooks, NetSuite, SAP)
8. Success metrics and guardrail metrics
9. Milestones and GTM plan

Level of detail: Comprehensive with specific numbers, timelines, and technical depth.

Save to: outputs/prd-lease-compliance.md
```

**💡 What's Happening Here?**

This prompt demonstrates 3 key Claude Code skills:
1. **Context Chaining:** Explicitly referencing multiple context layers (@outputs, @templates, @company-context)
2. **Prompt Engineering:** Specific requirements (what, how detailed) vs. vague "write a PRD"
3. **Template Customization:** Following template structure while adding domain-specific sections

Claude reads all your previous research and company context to generate a comprehensive PRD.

**Expected Output:** Complete human-readable PRD (~3000-4000 words) saved to outputs folder.

### Step 4: Add Citations and Cross-References

Now let's make the PRD traceable by adding citations to research findings:

**Prompt:**
```
Review the PRD you just created (outputs/prd-lease-compliance.md).

Add citations for all claims, data points, and user insights:

1. For market statistics: Add inline citations [Source: market-research.md:line_number]
2. For user pain points: Add citations [Source: user-research.md:line_number]
3. For user quotes: Include direct quotes from user interviews if available

Example citation format:
"Real estate legal teams manage an average of 500+ leases [Source: market-research.md:45], spending 40 hours/month on manual compliance reporting [Source: user-research.md:78]."

Ensure every key claim is traceable to source research.

Update outputs/prd-lease-compliance.md with citations.
```

**💡 What's Happening Here?**

This demonstrates **Cross-Referencing & Citation Management:**
- Creating evidence-based, traceable documents
- Stakeholders can verify claims by checking source line numbers
- Critical for enterprise PM work and gaining stakeholder trust

**Expected Output:** PRD with inline citations linking every claim to specific research findings.

### Step 5: Refine Specific Section (Iterative Editing)

Let's improve the Technical Specifications section without rewriting the entire PRD:

**Prompt:**
```
I want to expand the Technical Specifications section only.

For the Technical Specifications section in outputs/prd-lease-compliance.md:

Expand the Integration Points subsection to include:
- Specific accounting systems: QuickBooks (API v3), NetSuite (SuiteTalk REST), SAP (OData), Workday (REST API)
- Authentication methods: OAuth 2.0 for each system
- Data sync frequency options: Real-time (webhook), Hourly, Daily, Manual
- Error handling: Retry logic (3 attempts with exponential backoff), fallback to manual entry
- Rate limits: QuickBooks (500 req/min), NetSuite (1000 req/hour), SAP (custom), Workday (200 req/min)

Edit ONLY the Integration Points subsection - leave all other sections unchanged.
```

**💡 What's Happening Here?**

This demonstrates **Iterative Refinement & Section-Level Editing:**
- Refining specific sections without regenerating entire document
- Critical skill for production work - you rarely rewrite everything
- Saves time and preserves other sections that are already good

**Expected Output:** Enhanced Integration Points subsection with detailed technical specs.

## PART B: Generate Agent-Consumable PRD

### Step 7: Generate Agent PRD (JSON)

Now let's create the machine-readable version for automation systems:

**Prompt:**
```
Generate an agent-consumable PRD in JSON format based on the human PRD (outputs/prd-lease-compliance.md).

Follow the schema structure in @templates/prd-schema.json.

Include ALL sections from human PRD in structured format:

1. **metadata:** version, generated_date, human_prd_reference

2. **introduction:** problem_statement, key_insights, impact_sizing (structured objects)

3. **goals_and_metrics:** success_metrics[], guardrail_metrics[]

4. **user_stories[]:** Extract all user stories with structured fields:
   - persona, user_story, acceptance_criteria[], priority (P0/P1/P2), complexity (Low/Medium/High)

5. **nfrs (non-functional requirements):** Structure as typed objects:
   - performance: {response_time_ms, throughput, max_concurrent_users}
   - security: {authentication, encryption_at_rest, encryption_in_transit, compliance_standards[]}
   - availability: {uptime_sla_percent, rto_hours, rpo_hours, backup_frequency}
   - data_retention: {retention_period_years, deletion_policy, pii_handling}

6. **technical_specs:**
   - data_models: {Lease: {fields, types, relationships}, ComplianceReport: {...}, AuditTrail: {...}}
   - api_specs: {endpoints[], auth_method, rate_limits{}}
   - integrations: {systems[], sync_frequency, webhook_support}

7. **agent_workflow:**

   a) task_graph (DAG - Directed Acyclic Graph):
   ```
   Nodes:
   - lease_data_extraction (Agent: DataIngestionAgent, action: "Extract lease data from PDF/CSV")
   - data_validation (Agent: ValidationAgent, action: "Validate data completeness and format", depends_on: [lease_data_extraction])
   - compliance_check (Agent: ComplianceAgent, action: "Check IFRS/GAAP compliance", depends_on: [data_validation])
   - risk_scoring (Agent: RiskScoringAgent, action: "Calculate risk scores", depends_on: [data_validation])
   - report_generation (Agent: ReportAgent, action: "Generate audit report", depends_on: [compliance_check, risk_scoring])
   ```

   b) handoff_contracts[] - For each agent handoff, define:
   ```
   {
     "from_agent": "DataIngestionAgent",
     "to_agent": "ValidationAgent",
     "input_schema": {
       "lease_data": {"type": "array", "required_fields": ["lease_id", "tenant", "start_date", "end_date", "rent"]},
       "source_file": {"type": "string"}
     },
     "output_schema": {
       "validated_leases": {"type": "array"},
       "validation_errors": {"type": "array"}
     },
     "invariants": ["All lease_ids unique", "Dates in ISO8601"],
     "failure_modes": [
       {"condition": "Malformed input", "action": "Return error, halt pipeline"},
       {"condition": "Partial data", "action": "Log warning, process valid records"}
     ]
   }
   ```

   c) tool_manifest[] - List all tools:
   ```
   {
     "tool_id": "pdf_lease_extractor",
     "tool_name": "PDF Lease Data Extractor",
     "owning_agent": "DataIngestionAgent",
     "type": "API",
     "idempotent": true,
     "timeout_ms": 30000,
     "retry_strategy": "exponential_backoff",
     "fallback": "Manual upload form"
   }
   ```

8. **milestones:** Extract from human PRD, structure as objects with dates

9. **validation:** go_no_go_criteria[] as boolean checks
   - Example: {"criterion": "All P0 features complete", "must_pass": true}

Format: Pretty-printed JSON with 2-space indentation

Save to: outputs/prd-lease-compliance-agent.json
```

**💡 What's Happening Here?**

This creates a machine-readable PRD that:
- **Mirrors all human PRD content** in structured format
- **Adds agent workflow specifications** (task graph, handoff contracts, tool manifest)
- **Enables automation** - downstream agents can parse JSON directly without interpreting prose

**Key Concepts:**
- **Task Graph (DAG):** Shows which agent tasks can run in parallel vs. sequentially
- **Handoff Contracts:** Explicit schemas for agent-to-agent data transfer (prevents silent failures)
- **Tool Manifest:** Lists all tools with idempotency flags (critical for retry behavior)

**Expected Output:** Complete agent PRD JSON file (~500-800 lines).

### Step 8: Understanding Task Graphs (DAG)

Let's visualize the task graph to understand parallel execution:

**Prompt:**
```
Create a visual representation of the task graph from outputs/prd-lease-compliance-agent.json.

Generate a simple text-based flowchart showing:
- All agent nodes
- Dependencies (arrows)
- Which tasks can run in parallel

Save as: outputs/task-graph-visualization.md
```

**💡 What's Happening Here?**

**Task Graph Basics:**

Linear execution (sequential):
```
Extract → Validate → Compliance → Risk → Report
(Total time: Sum of all steps)
```

Task Graph (DAG with parallelization):
```
Extract Lease Data (DataIngestionAgent)
         ↓
Validate Data (ValidationAgent)
         ↓
    ┌────┴────┐
    ↓         ↓
Compliance  Risk Scoring
  Check    (can run in parallel!)
    ↓         ↓
    └────┬────┘
         ↓
  Generate Report
```

**Benefit:** Compliance Check and Risk Scoring can run simultaneously, reducing total execution time.

**Expected Output:** Visual task graph showing parallel execution opportunities.
